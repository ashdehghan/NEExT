Completed Sprint: Sprint 1 - Selection Hierarchy and Lineage Context

Goal

Make Workbench artifact selection explain the project lineage from dataset to feature to embedding to model, so the user can see what belongs to the selected context without losing the current navigation pattern.

Acceptance Criteria
  • Left Panel artifact lists preserve the active project or dataset scope.
  • Project selection shows project datasets only; downstream artifacts require a dataset or lower artifact branch.
  • The dataset list remains stable across branch selection so sibling datasets are still available.
  • Dataset, feature, embedding, and model selection keep sibling artifacts visible within the same dataset branch.
  • Feature and embedding selection mark related downstream artifacts instead of hiding unrelated same-dataset artifacts.
  • Existing click-to-navigate behavior remains: selecting an artifact opens that artifact kind's Center View.

Verification

  • npm exec tsc passed.
  • python3 -m pytest tests/test_workbench.py passed with 34 tests.
  • npm run test:e2e passed with 14 browser tests.
  • Manual browser verification at http://127.0.0.1:8765 confirmed stable dataset navigation and related markers with no console warnings or errors.

Lineage Scope Rules

Selected Context Left Panel Context To Show
Project All datasets in the active project. Features, embeddings, and models stay empty until a dataset, feature, embedding, or model establishes a coherent branch.
Dataset All project datasets remain visible with the selected dataset highlighted. Downstream sections show features whose source_dataset input points to it, embeddings built from those features, and models built from those embeddings.
Feature All project datasets remain visible with the feature source dataset highlighted. All sibling features, embeddings, and models from that dataset remain visible. Mark embeddings consuming the selected feature and models consuming those embeddings.
Embedding All project datasets remain visible with the embedding source dataset highlighted. All sibling features, embeddings, and models from that dataset remain visible. Mark models consuming the selected embedding.
Model All project datasets remain visible with the model source dataset highlighted. All sibling features, embeddings, and models from that dataset remain visible while the selected model is highlighted.

Current State

NEExT Core Library

  • Input and graph loading: CSV, DataFrame, URL, and NetworkX loading through graph IO workflows.
  • Graph structures: graph wrappers, graph collections, egonets, and k-hop decomposition.
  • Features: structural node feature computation and custom feature registration.
  • Embeddings: graph embedding computation, including Wasserstein-style embeddings.
  • ML and datasets: sklearn-compatible datasets, model training, metrics, and feature importance support.
  • Research utilities: synthetic generators, anomaly helpers, adapters, presets, and outlier utilities.

NEExT Workbench

  • Runtime: local single-user FastAPI backend with React release assets.
  • Workspace: project-first persisted workspace under ~/NEExT-Workbench.
  • Artifacts: datasets, features, embeddings, models, and jobs are persisted as project artifacts.
  • Execution: dataset preparation, feature execution, embedding execution, and model execution are real, local workflows.
  • Inspection: previews, analysis views, job records, command output, inspector context, and status surfaces are available.

Compute DAG

Workbench compute starts at configured Dataset artifacts. Downstream artifacts store typed input references so the lineage remains explicit.

Dataset

Configured from library source templates, then prepared into canonical Parquet plus manifests, mappings, stats, and logs.

Feature

One structural feature artifact targets one prepared dataset and writes feature output only on successful execution.

Embedding

One embedding artifact consumes one or more feature artifacts from the same dataset and writes graph-level embeddings.

Model

One model artifact consumes one or more embeddings from the same dataset and writes trained model files plus metrics.

UUID artifacts Relative paths only Single local worker Auto-runs upstream planned/failed work
